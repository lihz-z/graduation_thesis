CLRNet/CLRNet-FGM 实验任务清单（供 AutoDL 上的 Codex 直接读取）

一、总目标

需要围绕 CLRNet 基线与改进模型 CLRNet-FGM 产出完整实验资产，包括：
1. 最终可用的最佳模型权重 .pth
2. 论文可填写的定量结果表
3. 论文可插入的定性可视化图片
4. 消融实验结果
5. 部署前模型导出验证结果

二、必须产出的实验结果清单

1. 基线模型结果

需要训练或复现 CLRNet baseline，并输出：
1. 测试集整体指标
   - F1
   - Precision
   - Recall
2. 推理效率指标
   - 单张图像平均推理时间 ms
   - FPS
3. 模型复杂度
   - 参数量 Params
   - 模型大小 MB
4. 保存文件
   - best_baseline.pth
   - baseline 测试日志
   - baseline 配置文件副本

2. 改进模型最终结果

需要训练 CLRNet-FGM，并输出：
1. 测试集整体指标
   - F1
   - Precision
   - Recall
2. 与 baseline 的对比提升
   - ΔF1
   - ΔPrecision
   - ΔRecall
3. 推理效率指标
   - 单张图像平均推理时间 ms
   - FPS
4. 模型复杂度
   - 参数量 Params
   - 模型大小 MB
5. 保存文件
   - best_fgm.pth
   - 最终测试日志
   - 最终训练配置文件

3. 分场景实验结果

为了支撑论文中“雨天鲁棒性分析”，需要按场景拆分测试结果，至少包括：
1. 小雨场景
   - F1 / Precision / Recall
2. 中雨场景
   - F1 / Precision / Recall
3. 大雨场景
   - F1 / Precision / Recall
4. 强反光/积水场景
   - F1 / Precision / Recall
5. 遮挡明显场景
   - F1 / Precision / Recall

如果当前数据集没有完全细分，也至少要给出：
1. 普通雨天
2. 强干扰雨天

4. 消融实验结果

这是论文里必须有的。建议至少产出以下几组：

1. Baseline
   - CLRNet
2. 仅频域门控分支
   - CLRNet + Frequency Gate
3. 仅方向注意力分支
   - CLRNet + Directional Attention
4. 完整模块
   - CLRNet + FGM

每组都要输出：
1. F1
2. Precision
3. Recall
4. FPS
5. 参数量或模型大小

5. 模块参数敏感性实验

为了支撑第四章方法合理性，建议补这些：

1. 频域门控阈值实验
   - 不同 threshold 下的指标
2. 方向分支数量实验
   - 不同 direction_bins 下的指标
3. 模块插入位置实验
   - 不同 backbone/stage 插入位置下的指标

每个实验至少记录：
1. 参数设置
2. F1
3. Precision
4. Recall
5. FPS

6. 训练过程结果

为了生成训练曲线图，需要保存：
1. train loss 曲线数据
2. val loss 曲线数据
3. F1 随 epoch 变化曲线
4. 最优 epoch
5. 最优模型对应指标

建议输出成：
1. csv
2. json
3. png 曲线图

7. 定性可视化结果

论文里至少需要以下图片：

1. Baseline 检测结果图
2. 改进模型检测结果图
3. 两者对比图
4. 小雨场景对比图
5. 中/大雨场景对比图
6. 强反光场景对比图
7. 失败案例图
8. 边缘/细长结构恢复效果图

每类建议保存 3-5 张。

8. 失败案例分析结果

为了避免论文只展示成功样例，建议单独整理：
1. baseline 失败样例
2. 改进模型失败样例
3. baseline 失败但改进模型成功的样例
4. 改进模型仍失败的极端样例

每张图最好附：
1. 图像名
2. 场景类型
3. 失败原因简述

9. 部署前导出验证结果

为了支撑后续部署章节，需要保存：
1. best_fgm.pth
2. 导出的 onnx 文件
3. ONNX 推理成功记录
4. PyTorch 与 ONNX 输出一致性检查结果
5. 若继续做 TensorRT：
   - engine 文件
   - TensorRT 推理日志
   - TensorRT FPS

三、最终要形成的论文结果表

AutoDL 上的 Codex 最终应整理出这些表的数据源：

1. 基线与改进模型总体性能对比表
2. 分场景性能对比表
3. 消融实验结果表
4. 参数敏感性实验表
5. 部署测试结果表

四、最终要形成的论文图片

1. 训练收敛曲线图
2. Baseline vs 改进模型检测效果对比图
3. 不同雨天场景对比图
4. 失败案例分析图
5. 部署流程或部署效果图

建议让 Codex 按下面结构整理输出：

    curves/
    vis/
  fgm/

    curves/

    onnx_eval.json
    tensorrt_eval.json
  summary/

    final_metrics.csv
    final_figures/

六、建议直接交给 Codex 的任务要求

你可以让 AutoDL 上的 Codex 按这个目标执行：

1. 跑通 CLRNet baseline 训练、验证、测试并保存最佳权重
2. 跑通 CLRNet-FGM 训练、验证、测试并保存最佳权重
3. 输出总体指标、分场景指标、推理速度、模型大小
4. 完成最小消融实验
5. 导出训练曲线和定性可视化图片
6. 导出 ONNX，并验证输出一致性
7. 最终生成适合论文填写的 csv/xlsx/json/png

七、最核心的一句话

AutoDL 上的 Codex 不只是“把模型跑起来”，而是要一次性产出能直接回填论文第五章和第六章的完整实验资产。
